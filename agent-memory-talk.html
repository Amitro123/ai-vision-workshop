חלק 2

פסיכולוגיה של זיכרון — אדם מול מכונה

👤 בן אדם

זוכר קוד SMS לדקה — ומיד שוכח.

אבל יכול להיזכר תוך שניות בשיחה שהייתה לפני שלושה שבועות, כולל הקשר רגשי וציר זמן.

זיכרון = מה חשוב, לא רק מה קרה לאחרונה.

VS

🤖 רוב האייג'נטים

מחזיקים state זמני בתוך Context Window.

ברגע שהחלון מתמלא, מידע חדש דוחק החוצה מידע ישן.

אין סלקציה, אין עדיפויות, אין משמעות לאורך זמן.

💡 האנלוגיה: Context Window הוא שולחן עבודה זמני. Memory היא הספרייה, הארכיון, והיכולת לדעת מה לשלוף ומתי.

חלק 4 · שכבות הזיכרון

Working Memory × Long-term Memory

⚡ Working Memory

  • מצב השיחה הנוכחית
  • Message history פעיל
  • Session context ו-metadata
  • Summaries של threads ארוכים
  • Task progress וסטטוס זמני

🏛️ Long-term Memory

  • User preferences ו-facts
  • Important knowledge חוצה סשנים
  • Semantic retrieval
  • Institutional knowledge משותף
  • Promoted memories לפי policy
Memory בפרודקשן היא לא רק persistence — היא policy.
חלק 4 · Policy in Practice

Forgetting Rules — Config Snippet

# Redis Agent Memory — forgetting policy example memory_config = { "working_memory": { "ttl": 3600, # 1 hour session window "max_messages": 50, # then summarize + compact "summarize_on_overflow": True }, "long_term_memory": { "promote_threshold": "high_importance", "retention_policy": "user_preferences:forever", "forget_policy": "session_artifacts:7d", "shared_across_agents": ["team_context", "project_state"] }, "compliance": { "gdpr_delete_on_request": True, "pii_fields": ["email", "phone"] # auto-expire } }

לא רק "שמור הכל" — אלא הגדרה מפורשת של מה נשמר, כמה זמן, ואיפה.
